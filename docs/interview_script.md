# 面试讲解稿

## 30 秒项目简介

我做了一个 B 站爆款视频弹幕互动与评论情绪分析项目，分析对象是影视飓风的单条高播放视频。项目用 Python 采集公开视频基础指标、弹幕样本和评论样本，用 pandas 做清洗和指标构建，用 jieba 和 SnowNLP 做中文关键词与情绪分析，再把结构化结果写入 MySQL，最后用 Flask + Echarts 做成本地可演示的数据看板。这个项目主要展示我从数据采集、数据处理、SQL 建模到可视化表达的完整数据分析流程。

## 2 分钟完整讲解

这个项目的业务问题是：一条爆款视频为什么互动高，用户讨论集中在哪些时间段，评论区主要关注什么主题，整体反馈偏正向还是负向。

数据方面，我选择了影视飓风的一条公开视频，采集了三类数据：第一类是视频基础指标，比如播放量、点赞、投币、收藏、评论和弹幕数；第二类是公开弹幕样本，解析弹幕出现的视频时间，用 30 秒作为时间桶；第三类是公开评论样本，只保留评论文本、点赞数、回复数和时间，不保存用户身份字段。

清洗阶段，我用 pandas 处理缺失文本、空评论、URL、表情符号和无意义字符。弹幕数据按视频时间转换成分钟和 30 秒时间桶，评论和弹幕都做了文本清洗。文本分析方面，我用 jieba 做中文分词和停用词过滤，生成弹幕和评论关键词；情绪分析用 SnowNLP 生成 0 到 1 的情绪分数，并划分为正向、中性和负向。

指标设计方面，我计算了点赞率、投币率、收藏率、分享率、评论率、弹幕率和综合互动率；同时构建了弹幕高峰片段、关键词词频、评论情绪占比、评论情绪与点赞数关系等指标。

数据库方面，我设计了 video_info、danmaku_info、comment_info、interaction_metrics、danmaku_timeline、keyword_metrics 和 sentiment_metrics 等表，把清洗后的明细数据和聚合指标写入 MySQL。

最后我用 Flask 提供 API，用 Echarts 做了一个一页式看板，包含视频概览、互动率、弹幕时间轴、弹幕高峰片段、关键词和评论情绪。项目结论是：该视频综合互动率约 21.30%，弹幕高峰主要出现在开头、12 分钟附近和结尾附近；评论样本中正向情绪约 58.4%，说明整体反馈偏正向，但弹幕负向和吐槽比例也相对明显。

## 常见追问

### 为什么只分析一条视频？

这个项目定位是单视频深度分析，不是 UP 主全量经营分析。单视频更适合在短时间内把数据采集、清洗、文本分析、指标设计和可视化链路做完整，也方便面试时围绕一个具体案例讲清楚。

### 数据怎么来的？

数据来自 B 站公开视频公开接口，包括视频基础信息、公开弹幕 XML 和公开评论分页接口。项目没有使用 API Key，也没有依赖 Cookie。评论和弹幕只保留分析字段，不保存用户昵称、ID、头像等身份信息。

### 怎么处理评论接口风控？

评论采集优先使用 B 站 WBI 签名分页接口。脚本中做了分页、去重和异常停止处理。如果某一页触发风控，程序会保留已采集数据并停止，不会导致后续清洗和分析流程中断。

### 如何体现 Python 能力？

Python 部分包括 requests 采集、XML/JSON 解析、pandas 清洗、时间桶聚合、jieba 分词、SnowNLP 情绪分析、wordcloud 词云生成和一键 pipeline 串联，覆盖了数据分析岗位常见的 Python 使用场景。

### 如何体现 SQL / MySQL 能力？

项目不是只导出 CSV，而是设计了 MySQL 表结构，包括视频维度、弹幕明细、评论明细和指标表。脚本可以自动建库建表、清空并导入新结果，后续可以直接用 SQL 查询互动指标、关键词和情绪分布。

### 情绪分析准确吗？

我会明确说明这是轻量级探索性分析。SnowNLP 可以快速给出中文情绪倾向，但对反讽、梗、弹幕语境不一定准确。所以我不会把它包装成严格舆情模型，而是作为数据分析作品中的一个文本分析指标。

### 如果继续优化，会怎么做？

可以从三方面优化：第一，扩展到影视飓风多条视频，做选题对比；第二，结合视频章节或字幕，把弹幕高峰和具体内容片段对应起来；第三，用更适合社交媒体语料的中文情绪模型替换 SnowNLP，提高文本分析准确性。
